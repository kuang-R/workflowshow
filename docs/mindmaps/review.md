# 需求评审流程

## 阶段
- 需求提出
  - 产品经理
  - 业务方反馈
  - 数据分析
- 文档编写
  - 需求背景
  - 用户故事
  - 验收标准
- 技术评估
  - 可行性分析
  - 技术选型
  - 风险识别
- 评审会议
  - 参会人
    - 产品经理
    - 技术负责人
    - [前端](/mindmaps/frontend)
    - [后端](/mindmaps/backend)
    - 测试
  - 评审要点
    - 需求是否清晰
    - 技术是否可行
    - 排期是否合理
- 评审结果
  - 通过 → 拆分任务
  - 需修改 → 回退到文档编写
  - 驳回 → 重新评估

## 产出物
- 需求文档
- 技术方案
- 任务拆分清单
- 排期计划
