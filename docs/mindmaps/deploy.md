  # 发布上线流程

  ## 阶段
  - 代码合入
    - Feature 分支
    - Code Review
    - Merge 到主分支
  - CI 构建
    - 自动编译
    - 单元测试
    - 代码扫描
  - 测试环境
    - 自动部署
    - 功能测试
    - 回归测试
  - 预发环境
    - 灰度发布
    - 预发验证
    - 性能测试
  - 生产发布
    - 分批灰度
    - 线上监控
    - 回滚预案

  ## 涉及角色
  - [前端](./frontend)
  - [后端](./backend)
  - 测试工程师
  - SRE / DevOps
  - 产品经理

  ## 关键节点
  - 测试通过 → 准予发布
  - 预发验证 → 可生产发布
  - 线上异常 → 回滚或热修复
  - 全量上线 → 发布完成
